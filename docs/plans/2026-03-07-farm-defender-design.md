## 游戏概述

### 核心概念
- **游戏名称**: Farm Defender（农场保卫战）
- **主题**: 扮演美国农庄主人，保护农场免受野生动物骚扰
- **玩法类型**: 混合塔防 + 实时操作
- **视觉风格**: 2D 俯视视角 + 卡通渲染
- **目标平台**: 网页浏览器（HTML5）
- **目标受众**: 休闲游戏玩家
- **单局时长**: 5-15 分钟

## 技术架构

### 技术栈
- **游戏框架**: Phaser 3
- **语言**: TypeScript
- **构建工具**: Vite
- **部署**: GitHub Pages

### 架构分层
```
┌─────────────────────────────────────────┐
│ 游戏界面 (UI Layer)                     │
│ HUD | 商店 | 暂停菜单 | 游戏结束屏幕   │
├─────────────────────────────────────────┤
│ 游戏逻辑 (Game Layer)                   │
│ 敌人 AI | 塔楼逻辑 | 子弹物理 | 碰撞检测│
├─────────────────────────────────────────┤
│ 数据管理 (Data Layer)                   │
│ 关卡数据 | 敌人属性 | 武器配置 | 存档   │
└─────────────────────────────────────────┘
```

### 核心模块
1. **GameScene** - 主游戏场景，处理游戏循环
2. **EnemyManager** - 敌人生成、移动、状态管理
3. **TowerManager** - 塔楼放置、升级、射击
4. **ProjectileSystem** - 子弹物理、碰撞检测
5. **WaveSystem** - 敌人波次编排
6. **EconomySystem** - 金币管理、购买逻辑
7. **UIManager** - 界面交互

### 数据流
```
敌人进入 → 触发塔楼 → 发射子弹 → 碰撞检测 → 伤害计算 → 金币奖励
                                                      ↓
敌人死亡 → 波次进度更新 ← ← ← ← ← ← ← ← ← ← ← ← ← ←
```

## 关卡与进度

### 关卡设计
- **总计**: 10 个关卡
- **难度**: 递增设计
- **每关特点**: 不同地图布局和敌人组合
- **胜利条件**: 抵挡住所有波次敌人
- **失败条件**: 敌人到达终点超过 10 次

### 经济系统
- 初始资金：$300
- 击杀奖励：根据敌人类型 $10-$50
- 每波开始奖励：$50 + 波数×10
- 失败惩罚：失去 1 条生命，资金减半

### 升级系统
- 每座塔可升级 3 次
- 升级提升：
  - 伤害 +20%
  - 射程 +15%
  - 射速 +10%
- 满级塔解锁特殊能力（如机枪塔变为"火神模式"）

## 美术与音效

### 美术风格
- 明亮饱和的卡通色调
- 农场背景：谷仓、玉米地、栅栏、水井
- 敌人采用夸张可爱的设计
- 子弹和特效使用粒子系统

### 音效设计
- 背景音乐：乡村西部风格（班卓琴、口琴）
- 音效：枪声、爆炸、敌人叫声、金币声
- 语音：简单的"警告！"、"干得好！"等提示
